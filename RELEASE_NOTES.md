# 版本更新（Release Notes）

## 2026-07-20

**取消儲存機制整個換掉：不再捲動 /saved，改為逐篇開分頁執行**（userscript 0.5.4）。

- **為什麼換**：Threads 的 /saved 收藏列表在伺服器端限制了往下捲動可載入的深度（實測約三週內），捲動巡覽永遠摸不到較舊的收藏；加上收藏列表是虛擬捲動、DOM 隨時抽換，點擊驗證極不可靠——舊機制連續多輪只能取消個位數貼文。
- **新機制**：面板「取消儲存」→ 選 `unsave.json` → 腳本逐篇用新分頁開啟清單內的貼文網址，在貼文頁點「⋯ → 取消儲存」，處理完自動關閉分頁換下一篇。不依賴捲動、不受列表深度限制，每篇結果（已取消／跳過／失敗）即時回報在面板上。
- **不誤殺**：只處理 `unsave.json` 明列的貼文；本地關鍵詞候選只標亮提示，永不自動取消。若貼文選單顯示「儲存」（代表其實沒被收藏，例如先前已取消過），自動跳過不動作——重複執行安全。
- **保險機制**：連續 5 篇失敗自動中止（防被限流時硬撞）；執行中按鈕變「停止逐篇取消」可隨時喊停；抓取起跑時若偵測到上一次的舊資料會先提醒清空，避免已取消的貼文再次進入待取消清單。
- **需要的一次性設定**：在 Chrome 允許 `www.threads.com` 的彈出式視窗（首次執行被擋時面板會提示）。
- 疑難排解：面板「debug log」可將每篇執行結果寫入 NDJSON 檔，回報問題時附上即可快速定位。

## 2026-07-18

- **瀏覽器面板防誤觸重構**（userscript 0.4.1）：移除「手動工具」十顆按鈕與 Auto AI Sync 自動輪詢面板，改為單一「取消儲存」大按鈕。每次按下都重新選擇 `unsave.json`——選檔動作本身就是「用的是最新分類結果」的確認，避免拿舊檔誤執行取消儲存。流程：選檔 → 載入並標亮 → 自動勾選建議取消 → confirm 對話框確認後執行。另修兩個既有 bug：歷次累積的「已處理」排除清單可能因 Threads 虛擬捲動被誤標，導致候選被無聲跳過（現在每次選檔即重置）；執行結束後標亮同步迴圈未關閉，造成頁面持續閃爍（現在流程結束自動清除標亮）。
- **安裝簡化為一行指令**（Windows：`irm … | iex`；macOS：`curl … | bash`）：自動 clone、建立正確版本的 Python 環境（系統裝什麼版本都不影響）、安裝依賴與瀏覽器、產生設定檔範本。手動安裝步驟仍保留在 README。
- 新增 macOS 雙擊啟動器 `run_classify.command`，與 Windows 的 `run_classify.cmd` 對應——兩個平台都是雙擊即跑完整流程。
- 安裝說明（中英 README）拆成 Windows 與 macOS 兩版，SOP、疑難排解、流程圖同步更新。

## 2026-07-14

**存回應也能保留完整脈絡了。** 過去收藏的若是一則回應，筆記只有那一則的內容；現在：

- 筆記新增「上文脈絡」區：從母帖到你收藏那則回應的完整對話線，逐則標注作者。
- 筆記新增可摺疊的「回覆區」（Obsidian callout，預設收合）：原帖作者的回覆全數保留，其他留言自動過濾太短或純 emoji 的內容。
- 分類與標題生成會一併參考上文，存回應時的誤分類明顯減少。
- 筆記 frontmatter 新增 `saved_kind` 欄位，標記收藏的是原帖（`root`）還是回應（`reply`）。
- 新設定區塊 `thread-context`（`config.json`）：可整組關閉回到舊行為，或調整回覆的字數門檻與收錄串數上限（一串對話可含多則；原帖作者的回覆一律全數保留，不受上限影響）。

已知限制：回覆區只收匿名瀏覽可見的回覆，深層回覆與登入牆後的內容不會收錄。
